Managed Agent Workspaces are the planned business-facing layer for teams that
want an agent workforce without turning each employee into a systems
administrator. A workspace would let an authorized operator provision
role-specific agents—such as a lead-research agent, customer-service agent, or
competitor-tracking agent—inside one governed company boundary.

## The job

The product is designed to replace a brittle collection of individually hosted
agents, personal API keys, chat-platform tokens, and undocumented configuration.
The intended team layer includes SSO, role-based access, centralized guardrails,
workspace-level cost visibility, backups, audit history, and one clean invoice.

## The boundary

Each workspace keeps company context, credentials, authority, and execution
evidence separate. External actions remain capability-bound and approval-aware;
the catalogue does not promise that every integration or action can run without
human review. A managed workspace is not a shared root login and is not a resale
of an unmanaged server.

## Availability

This offer is planned and not yet live or generally available. Team SSO,
centralized guardrails, billing consolidation, uptime, and customer outcomes
still require implementation and runtime proof before the catalogue can advance
its status. Read the [managed agent infrastructure operating
thesis](https://github.com/fongryan/portfolio/blob/main/docs/managed-agent-infrastructure.md).
